🔧 Depannaj Rezo

Aprann kijan depane pwoblèm rezo komen yo ak kòmand ping, tracert, ipconfig, nslookup.

Metòd Depannaj Rezo

Bon depannaj swiv yon pwosesis sistematik — pa devine okouran. Kòmanse pi ba (Fizik) rive pi wo (Aplikasyon).

1

Idantifye Pwoblèm nan

"Ki egzakteman ki pwoblèm? Ki aparèy? Depi kilè? Èske lòt moun afekte?"

2

Etabli Teyori

Panse sou koz posib. Kòmanse ak pi senp: Kab konekte? IP konfigire? DNS ap travay?

3

Tès Teyori a

Ping, tracert, ipconfig, nslookup — itilize zouti yo pou konfime oswa rejte teyori ou.

4

Reyalize Plan

Yon fwa ou idantifye pwoblèm nan, reyalize yon plan pou rezoud li epi aplike li.

5

Verifye ak Dokimante

Verifye solisyon an travay, avize itilizatè yo, epi dokimante sa ou fè.

Pwoblèm Komen yo ak Solisyon

Verifye an lòd:

  1. Kab fizik konekte? Limyè LAN sou switch ak routeur?
  2. Ping gateway: ping 192.168.1.1 — si echwe, pwoblèm lokal
  3. Ping DNS: ping 8.8.8.8 — si echwe, pwoblèm routing oswa ISP
  4. Ping non domèn: ping google.com — si echwe men 8.8.8.8 travay, pwoblèm DNS

Konsomasyon APIPA (169.254.x.x): Aparèy pa ka jwenn sèvè DHCP. Renouvle bail DHCP:

# Windows
ipconfig /release
ipconfig /renew

# Linux
sudo dhclient -r eth0
sudo dhclient eth0

Si ou ka ping IP men pa non domèn — pwoblèm DNS. Solisyon:

# Netwaye cache DNS (Windows)
ipconfig /flushdns

# Chanje DNS manyèlman pou Google
DNS Prefere: 8.8.8.8
DNS Altènatif: 8.8.4.4

Solisyon:

  • Deplase pi prè access point
  • Chanje kanal Wi-Fi (evite 1, 6, 11 si vwazen yo itilize yo)
  • Mete routeur nan kote santral, wo (pa bò atè)
  • Redémarrer access point
  • Chanje frekans (essaye 5 GHz si aparèy la sipòte)

Kòmand Depannaj Enpòtan

ping

Teste si yon aparèy aksesib sou rezo. Itilize ICMP.

Command Prompt / Terminal

C:\> ping google.com

Pinging google.com [142.250.80.46] with 32 bytes of data:

Reply from 142.250.80.46: bytes=32 time=18ms TTL=117

Reply from 142.250.80.46: bytes=32 time=20ms TTL=117

Ping statistics: Sent = 4, Received = 4, Lost = 0 (0% loss)

ipconfig / ifconfig

Montre konfigirasyon IP aparèy ou.

Command Prompt

C:\> ipconfig /all

Ethernet adapter:

  IPv4 Address. . . : 192.168.1.105

  Subnet Mask . . . : 255.255.255.0

  Default Gateway . : 192.168.1.1

  DNS Servers . . . : 8.8.8.8

  Physical Address. : 00-1A-2B-3C-4D-5E

tracert / traceroute

Montre chak étape (hop) pakè a pase pou rive nan destinasyon an.

Command Prompt

C:\> tracert google.com

1 <1ms 192.168.1.1 ← Routeur lokal (gateway)

2 8ms 10.0.0.1 ← Routeur ISP

3 15ms 203.x.x.x ← Nan ISP la

4 18ms 142.250.80.46 ← Sèvè Google

nslookup

Rekèt DNS manyèl — tradui non an IP oswa IP an non.

Command Prompt

C:\> nslookup google.com

Server: dns.google (8.8.8.8)

Name: google.com

Address: 142.250.80.46

netstat

Montre koneksyon rezo aktif ak pò k ap koute.

Command Prompt

C:\> netstat -an

Proto Local Address Foreign Address State

TCP 0.0.0.0:80 0.0.0.0:0 LISTENING

TCP 192.168.1.5:52432 142.250.80.46:443 ESTABLISHED

arp

Montre oswa jere tab ARP — korèlasyon IP ak MAC address.

Command Prompt

C:\> arp -a

Interface: 192.168.1.5

192.168.1.1 00-1a-2b-3c-4d-5e dynamic ← Gateway

192.168.1.10 00-aa-bb-cc-dd-ee dynamic
