🧪 Labs Pratik Rezo

10 labs pratik pou mete konesans ou an pratik — soti nan baz jiska konfigirasyon avanse.

🎯
Konsèy:

Fè chak lab nan lòd yo. Chak youn baze sou konesans lab anvan an. Ou ka fè yo nan Cisco Packet Tracer, VirtualBox, oswa materyèl reyèl.

Lab 1

Idantifye Aparèy Rezo yo

Debitan

🎯 Objektif

Idantifye diferan aparèy rezo ak wòl yo nan yon dyagram.

🛠️ Zouti Nesesè

  • Papye ak kreyon oswa Packet Tracer
  • Dyagram rezo ki bay nan paj Aparèy

📋 Etap yo

  1. Gade dyagram rezo a
  2. Idantifye chak aparèy: Router, Switch, Hub, PC, Sèvè
  3. Ekri wòl chak aparèy ak kouch OSI yo opere nan
  4. Yon Router = Kouch 3, Switch = Kouch 2, Hub = Kouch 1

✅ Rezilta Espere

Ou ka idantifye kòrèkteman chak aparèy ak wòl li.

💬 Kesyon Refleksyon

  • Ki diferans ant Switch ak Hub?
  • Poukisa Router opere nan Kouch 3?
📸

Kaptire ekran / foto dyagram ou idantifye a

Lab 2

Konekte PC a Switch ak Router

Debitan

🎯 Objektif

Bati yon rezo debaz ak 2 PC, 1 switch, ak 1 routeur nan Packet Tracer.

🛠️ Zouti

  • Cisco Packet Tracer (gratis — netacad.com)

📋 Etap yo

  1. Ouvri Packet Tracer, ajoute: 2 PC, 1 Switch 2960, 1 Router 1941
  2. Konekte PC1 → Switch (pò Fa0/1) ak PC2 → Switch (pò Fa0/2)
  3. Konekte Switch (pò Fa0/24) → Router (pò Gig0/0)
  4. Konfigire IP sou PC1: 192.168.1.10/24, Gateway: 192.168.1.1
  5. Konfigire IP sou PC2: 192.168.1.20/24, Gateway: 192.168.1.1
  6. Konfigire Router entèfas: ip address 192.168.1.1 255.255.255.0

✅ Rezilta Espere

PC1 ka ping PC2 ak Router.

💬 Kesyon Refleksyon

  • Poukisa Gateway enpòtan?
  • Ki sa ki ta pase si masque sou-rezo diferan?
Lab 3

Konfigire Adrès IP Manyèlman

Debitan+

🎯 Objektif

Konfigire adrès IP estatik sou Windows ak Linux manyèlman.

📋 Etap yo (Windows)

# Metòd 1: GUI
Network Settings → Change adapter options → Properties → IPv4
IP: 192.168.1.50, Masque: 255.255.255.0, Gateway: 192.168.1.1

# Metòd 2: Kòmand (Admin)
netsh interface ip set address "Ethernet" static 192.168.1.50 255.255.255.0 192.168.1.1

📋 Etap yo (Linux)

# Tanporèrman
sudo ip addr add 192.168.1.50/24 dev eth0
sudo ip route add default via 192.168.1.1

# Pèmanan (Ubuntu/Debian — /etc/netplan/)
network:
  ethernets:
    eth0:
      addresses: [192.168.1.50/24]
      gateway4: 192.168.1.1

💬 Kesyon Refleksyon

  • Diferans ant IP estatik ak DHCP?
  • Ki aparèy ki dwe toujou gen IP estatik?
Lab 4

Tès Koneksyon ak ping

Debitan+

🎯 Objektif

Itilize ping pou tès koneksyon nan diferan nivo.

📋 Etap yo

# Etap 1: Teste loopback (stack TCP/IP lokal)
ping 127.0.0.1

# Etap 2: Teste adrès IP pa ou
ping 192.168.1.50

# Etap 3: Teste gateway (routeur)
ping 192.168.1.1

# Etap 4: Teste sèvè DNS piblik
ping 8.8.8.8

# Etap 5: Teste nom domèn (DNS)
ping google.com

✅ Rezilta Espere

Tout 5 tès reisi. Si Etap 4 travay men 5 echwe — pwoblèm DNS.

💬 Kesyon Refleksyon

  • Si ping 127.0.0.1 echwe, ki pwoblèm sa montre?
  • Si Etap 3 echwe men Etap 2 travay, ki pwoblèm sa?
Lab 5

Trace Wout ak tracert

Entèmedyè

🎯 Objektif

Itilize tracert/traceroute pou idantifye chak hop nan wout yon pakè.

📋 Etap yo

# Windows
tracert google.com

# Linux/Mac
traceroute google.com

# Analiz chak liy:
1 <1ms 192.168.1.1 ← Routeur lokal
2 10ms 10.x.x.x ← ISP
3 *.* * * * ← Hop ki pa reponn (pa nòmal)

💬 Kesyon Refleksyon

  • Poukisa kèk hop montre * * *?
  • Ki valè TTL la fè?
Lab 6

Rekèt DNS ak nslookup

Entèmedyè

🎯 Objektif

Itilize nslookup pou verifye rekò DNS.

📋 Etap yo

# Rekèt debaz
nslookup google.com

# Rekèt type MX (rekò imel)
nslookup -type=MX gmail.com

# Rekèt avèk yon sèvè DNS espesifik
nslookup google.com 8.8.8.8

# Reverse DNS lookup (IP → Nom)
nslookup 8.8.8.8

💬 Kesyon Refleksyon

  • Ki rekò DNS ki retounen pou nslookup -type=MX gmail.com?
  • Poukisa rekèt avèk 8.8.8.8 bay menm rezilta?
Lab 7

Kreye VLAN Konseptyèlman

Avanse

🎯 Objektif

Konfigire VLAN sou yon switch Cisco nan Packet Tracer.

📋 Etap yo

# Kreye VLAN sou Switch Cisco
Switch> enable
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name IT
Switch(config)# vlan 20
Switch(config-vlan)# name Finance

# Asiye pò bay VLAN (Access Port)
Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

# Verifye VLAN
Switch# show vlan brief

💬 Kesyon Refleksyon

  • Poukisa PC nan VLAN 10 pa ka ping PC nan VLAN 20?
  • Ki sa ki nesesè pou kominikasyon ant VLAN yo?
Lab 8

Egzèsis Subnetting Pratik

Avanse

🎯 Objektif

Kalkile 4 sou-rezo soti nan 192.168.5.0/24 pou 4 depatman.

📋 Pwoblèm

Depatman yo bezwen:

  • IT: 60 aparèy
  • Finansye: 25 aparèy
  • RH: 10 aparèy
  • Manasman: 5 aparèy

✅ Solisyon

# IT: 60 aparèy → /26 (62 max)
Rezo: 192.168.5.0/26 Òs: .1-.62 Broadcast: .63

# Finansye: 25 aparèy → /27 (30 max)
Rezo: 192.168.5.64/27 Òs: .65-.94 Broadcast: .95

# RH: 10 aparèy → /28 (14 max)
Rezo: 192.168.5.96/28 Òs: .97-.110 Broadcast: .111

# Manasman: 5 aparèy → /29 (6 max)
Rezo: 192.168.5.112/29 Òs: .113-.118 Broadcast: .119
Lab 9

Depane "Pa Gen Entènèt"

Entèmedyè

🎯 Objektif

Suiv yon pwosesis sistematik pou depane koneksyon Entènèt ki pa fonksyone.

📋 Etap yo

# 1. Wè konfigirasyon IP
ipconfig /all
# Verifye: IPv4, Masque, Gateway, DNS

# 2. Teste loopback
ping 127.0.0.1

# 3. Teste gateway
ping 192.168.1.1

# 4. Teste DNS piblik
ping 8.8.8.8

# 5. Teste DNS name resolution
ping google.com

# 6. Si DHCP pa mache
ipconfig /release
ipconfig /renew

💬 Kesyon Refleksyon

  • Si etap 3 echwe men 2 travay, ki pwoblèm?
  • Si etap 4 travay men 5 pa, ki koz posib?
Lab 10

Sekirize Rezo Wi-Fi Kay ou

Avanse

🎯 Objektif

Konfigire yon rezo Wi-Fi kay la pou maksimize sekirite.

📋 Etap yo

  1. Chanje mo de pase defaut admin routeur la (192.168.1.1 oswa 192.168.0.1)
  2. Chanje SSID a — pa itilize non ou, adrès ou, oswa "default"
  3. Aktive WPA3 (oswa WPA2-AES si WPA3 pa disponib)
  4. Chanje mo de pase Wi-Fi — minimòm 12 karaktè
  5. Aktive Rezo Envite pou vizitè
  6. Dezaktive WPS (vulnerability)
  7. Mete ajou firmware routeur la
  8. Aktive firewall routeur la

💬 Kesyon Refleksyon

  • Poukisa WPS se yon risk sekirite?
  • Poukisa rezo envite enpòtan?
